# #실시간 Live Feed UX Design

## Goal

한국 웹 베타의 첫 화면을 지도앱이 아니라 "지금 가도 되는지 10초 안에 판단하는 생활 피드"로 만든다.

## Scope

- 홈은 검색과 방금 올라온 현장 피드를 최우선으로 둔다.
- 사진 있는 제보와 사진 없는 제보를 다른 카드로 표시한다.
- "지금 많이 확인하는 곳"을 내부 인기 신호 기반으로 노출한다.
- 장소 상세는 판단, 신뢰 근거, 최근 제보, 행동 전환 순서로 재정리한다.
- 제보 작성은 빠른 입력을 유지하고 병원/관공서는 대기/주차 중심으로 보수적으로 다룬다.
- 물어보기/답변 루프는 `현장 답변하기 +2`와 완료 피드백을 더 선명하게 만든다.

## Product Positioning

`#실시간`은 네이버지도/카카오맵과 같은 지도 검색 앱이 아니라, 지도앱이 약한 "방금 현장 사진, 지금 줄, 지금 주차, 지금 혼잡"을 보여주는 현장 피드다.

## Home Design

홈 첫 화면은 다음 순서로 구성한다.

1. 검색창: `지금 어디가 궁금하세요?`
2. 추천 검색어: `울산 태화강`, `광안리 주차`, `황리단길 줄`처럼 초기 지역과 상황을 함께 보여준다.
3. 방금 올라온 현장: 사진 있는 제보는 큰 현장 사진 카드로 표시한다.
4. 사진 없는 상태 제보: 사진 카드처럼 보이지 않게 별도 상태 카드로 표시한다.
5. 지금 많이 확인하는 곳: 최근 3시간 내부 신호 기반 인기 장소를 보여준다.
6. 내 주변 답변 가능한 질문: `현장 답변하기 +2` 참여 루프를 유지한다.
7. 지도/혼잡/한산 섹션: 탐색 보조 영역으로 배치한다.

## Popular Places Signal

초기 베타는 외부 실시간 검색어 API가 아니라 내부 신호를 사용한다.

- 최근 제보 수
- 최근 질문 수
- 답변 대기 질문 수
- 사진 제보 수
- 홈/상세 노출 가능한 신뢰 정보
- 길찾기 CTA 클릭 수는 이후 서버 이벤트로 확장한다.

문구는 `실시간 인기 장소`보다 `지금 많이 확인하는 곳`을 기본으로 사용한다. 내부 신호 기반이라는 점에서 더 정직하고, 이후 TMAP 장소 혼잡도 같은 외부 신호를 붙여도 자연스럽다.

외부 API 검토 결과:

- 네이버 데이터랩은 검색어 추이 API라 실시간 순위 API로 쓰기 어렵다.
- 카카오내비는 길찾기/목적지 공유와 소요시간 보강에 적합하지만 공개 인기 목적지 순위 API는 확인되지 않았다.
- TMAP은 장소 혼잡도, 방문자 수, 유입력, 주변 인기 장소 제품군이 있어 P1 외부 보강 신호 후보로 둔다.

## Place Detail Design

장소 상세는 `판단 + 신뢰 + 행동` 화면으로 구성한다.

1. 현재 판단 카드: `제보 기준 괜찮음/애매/혼잡`, 사람, 줄, 주차 요약을 최상단에 둔다.
2. 신뢰 근거 카드: 마지막 현장 인증, 최근 3시간 제보, 사진 있는 제보, 인증 없는 제보를 보여준다.
3. 최근 제보 분리: 현장 인증 제보와 인증 없는 제보를 분리하고, 사진 없는 제보는 상태 카드로 표시한다.
4. 하단 CTA: `여기 지금 어때요?`, `지금 상황 알려주기`를 강하게 둔다.
5. 길찾기 CTA: `카카오내비로 길찾기`, `티맵으로 길찾기`를 제공하고 이후 내부 인기 점수 이벤트로 연결할 수 있게 한다.

## Report Design

제보 작성은 10초 입력을 목표로 한다.

- 일반 장소는 사진, 사람, 주차, 한 줄 코멘트를 우선한다.
- 줄과 날씨는 접힌 선택 입력으로 유지한다.
- 보상 정책은 `현장 인증 + 사진: +2`, `현장 인증만: +1`, `인증 없음: 보상 없음`으로 압축한다.
- 병원/관공서는 사진보다 대기, 주차, 접수 줄 중심으로 안내한다.
- 병원/관공서 사진 CTA는 `외부/주차장 사진만`으로 표현한다.
- 위치 실패 시 인증 없는 제보는 허용하되 낮은 신뢰도와 보상 없음이 분명해야 한다.

## Question And Answer Loop

- 물어보기 작성 후 `근처 사용자에게 물어보는 중` 상태가 장소 상세 질문 영역에 즉시 보인다.
- 홈의 답변 가능한 질문 카드는 장소명, 질문 유형, 작성 시간, `현장 답변하기 +2`를 보여준다.
- 답변 모드에서는 질문 내용을 상단에 고정하고 현장 인증 필요성을 명확히 알린다.
- 답변 완료 후 `답변 완료! 물어보기권 +2를 받았어요.` 수준의 강한 성공 피드백을 제공한다.

## Testing

- UX 카피 테스트는 홈 구조, 상태 카드, 인기 장소, 길찾기 CTA, 답변 완료 피드백을 검증한다.
- JS/TS 수정 후 `npm test`를 실행한다.
- 최종 검증은 가능하면 `pnpm verify`로 lint, typecheck, test, build까지 확인한다.
